# NEURA for Microsoft 365 (WIP)

**Status:** work in progress. Not released.

This page is the home for the MS365 track inside
[neura-office](https://github.com/ianustec/neura-office). Design notes, beta
updates and links will land here as the add-in takes shape.

## Positioning

A **private alternative to Microsoft Copilot** for Word, Excel, PowerPoint and
Outlook: same place in the UI, different brain.

| | Copilot | NEURA for Microsoft 365 |
|---|---|---|
| Apps | Word, Excel, PowerPoint, Outlook | Same |
| Backend | Microsoft Copilot services | **Your Open WebUI** |
| Models, RAG, tools, policies | Vendor | Yours |
| File generation | Microsoft stack | NEURA Office engines (`.docx` / `.pptx` / `.xlsx`) |
| Data | Flows through Microsoft Copilot | Stays on your Open WebUI deployment |

## What it will do

- Side-panel assistant inside Word, Excel, PowerPoint and Outlook
- Draft, rewrite and generate native Office files without leaving the app
- Call the same generators already live as Open WebUI tools
- Stay compatible with self-hosted / private Open WebUI setups

## Engines (already live in chat)

| App | Engine |
|-----|--------|
| Word | [Generate Documents](https://github.com/ianustec/openwebui-generate-documents) |
| PowerPoint | [Generate Slides](https://github.com/ianustec/openwebui-generate-slides) |
| Excel | [Generate Spreadsheets](https://github.com/ianustec/openwebui-generate-spreadsheets) |
| Outlook | Planned (draft / reply / attach generated files) |

## Why it belongs in this repo

Open WebUI tools prove generation quality in chat.
The MS365 add-in brings that quality **into the apps**, without sending company
data to a third-party Copilot stack. Tracking it here keeps the suite story in
one place: chat tools now, in-app assistant next.
